### Этап 1: Финансы (Finance) — Read Only
**Цель:** Дать пользователю понимание о движениях средств и выплатах.
1.  **Service**: Реализовать получение списка транзакций `/v1/finance/transaction/list` (v3).
2.  **View**: Отчет "Последние транзакции" и "Сводка начислений за период".
3.  **Router**: Команда `/finance` и меню.

### Этап 2: Операции (Operations) — Read Only (MVP)
**Цель:** Контроль цен и базовой информации о товарах.
1.  **Service**: Реализовать получение цен `/v1/product/info/prices`.
2.  **View**: Отчет "Текущие цены" (Цена, Sanctions, Discount).
3.  **Router**: Команда `/prices`.

### Этап 3: Маркетинг (Marketing) — Read Only (MVP)
**Цель:** Обзор рекламных кампаний.
*Важно:* Ozon Performance API иногда требует отдельной авторизации, но попробуем базовые методы Seller API.
1.  **Service**: Получение списка кампаний `/v1/performance/campaigns` (или аналог).
2.  **View**: Список активных кампаний и их статус.
3.  **Router**: Команда `/marketing`.

### Этап 4: Интеграция
1.  Обновить Главное Меню (`menu.py`).
2.  Подключить новые роутеры в `bot.py`.
3.  Обновить `settings.py` (если нужны новые права/токены).
